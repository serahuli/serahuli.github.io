Tip：Vue3.0 采用 TS重写

Vue3.0 六大亮点

  • Performance: 性能比Vue2.X快1.2 ~ 2倍
  • Tree shaking support: 按需编译， 体积比Vue2.X更小
  • composition API : 组合API（类似于 React Hooks）
  • Better TypeScript Support: 更好的TS支持
  • Custom Renderer API: 暴露了自定义渲染API
  • Fragment, teleport(Protal),Suspense: 更先进的组件

Vue3.0是怎么变快的

  1. diff方法优化

    Vue2.X是全量比较，Vue3.0新增了静态标记（PatchFlag）,当虚拟DOM更新之后，只对比带有静态标记的变量，并且通过 flag可以知道需要对比的类型（节点，样式 ext…）

    Vue3编译结果

    Vue2.X编译结果

  2. hoistStatic静态提升

    • Vue2.x无论元素是否参与更新，都会重新创建，然后再进行渲染
    • Vue3.0对于不参与更新的元素，会做静态提升，只会被创建一个，渲染时候直接使用

  3. cacheHandlers事件侦听器缓存

    • 默认 onClick会被视为动态绑定，每次都需要跟踪变化
    • 但是同一个函数，无需跟踪，只需要缓存使用就可以了

  4. SSR渲染

    当有大量静态的内容时候，这些内容会被当做纯字符串推进一个buffer里面，即使存在动态的绑定，会通过模板插值嵌入进去。这样会比通过虚拟dmo来渲染的快上很多很多。当静态内容大到一定量级时候，会用_ createStaticVNode方法在客户 端去生成一个static node,这些静态node,会被直接innerHtml,就不需要创建对象，然后根据对象渲染。
